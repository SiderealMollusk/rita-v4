# Research

Purpose:
1. hold time-bound investigation notes that inform plans or implementation
2. capture source-backed technical tradeoffs before they become policy or runbooks

What belongs here:
1. focused research memos with links to primary sources
2. integration comparisons and capability checks
3. recommendations that may later feed `docs/plans/` or `docs/requirements/`

What does not belong here:
1. finalized architecture decisions (move to requirements/prd)
2. execution logs (use `docs/progress_log/`)
3. operator step-by-step procedures (use runbooks/scripts and their READMEs)

How to add a new note:
1. create the next numbered file (`000N-topic.md`)
2. include date/context, findings, and explicit source links
3. keep conclusions falsifiable and scoped to current repo goals
