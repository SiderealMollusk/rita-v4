# new scripts & status checks
- added a bunch of scripts under scripts/1-session and scripts/2-ops/local for cluster lifecycle and verification
- 01-load-variables.sh: loads project/user context, warns if missing
- 02-docker-check.sh: checks if /var/run/docker.sock is present, fixes permissions, verifies docker ps works (no sudo)
- 03-k8s-up.sh: wakes up the cluster, ensures docker is reachable, starts k3d cluster, refreshes kubeconfig
- 04-k8s-status.sh: just runs k3d cluster list rita-local for a quick status
- 05-k8s-down.sh: stops the cluster, reclaims resources
- chmod-all-sh.sh: makes all scripts executable (handy after git clone)
- rebuild-cluster.sh: wipes and recreates the cluster, with guardrails so you don't nuke the wrong thing

these scripts make it way easier to check if docker/k3d are alive, and to bring the cluster up/down or rebuild it without manual steps
the docker check script is especially useful for catching socket/permission issues early
status script is a quick way to see if k3d thinks the cluster is up, even if the helper says otherwise
