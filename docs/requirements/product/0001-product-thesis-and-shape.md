# Product Requirement 0001

## Purpose

This document captures the top-level product shape.

It is meant to keep future design and implementation work anchored on the current thesis without pretending the system is fully specified.

## Product Thesis

The product is a software collective.

Humans, agents, and contributed compute work together through a shared office-like workspace.

The workspace should feel:

- socially legible
- visibly active
- accountable
- collaborative

not like an opaque automation backend.

## Current Shape

The current product shape has three visible layers:

1. a human-facing office
2. a machine coordination core
3. contributed compute and pilot surfaces

This is not a consumer chat app, a generic agent dashboard, or a pure automation backend.

It is a collective work environment.

## Scope Of This Requirement Set

The numbered product requirements in this folder are intentionally light.

They should answer:

- what the product is
- what the core nouns are
- where the major boundaries are
- what future implementation work must preserve

They should not try to fully specify runtime behavior, schemas, or APIs.
