# Product Requirement 0005

## Purpose

This document captures the current tentpole constraints that future implementation work should preserve.

## Agent Behavior Constraint

Agents should behave like diligent office workers:

- they move cards
- write docs
- leave notes
- negotiate in Talk
- create visible movement

The system is not just agent output pasted into a UI.

## Trust Constraint

The current product model assumes trusted contributors.

The system should prioritize:

- visibility
- attribution
- social accountability

over an overstated promise of hard internal secrecy.

## Product Constraint

The product should remain understandable as:

- a collective
- with a front office
- with a machine core
- with contributed compute

Future implementation work should avoid collapsing everything into a generic chatbot platform or a pure backend workflow engine.
