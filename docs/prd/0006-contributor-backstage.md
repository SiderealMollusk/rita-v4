# 0006 - Contributor Backstage

## Shape

The backstage stack is:

- `Hive <-> contributor server <-> contributor page`

## Purpose

This is where contributors manage:

- compute resources
- pilot licenses
- teams
- souls
- castes
- characters
- budgets
- visibility into usage

## Compute Resources

Suggested fields:

- endpoint
- auth or key reference
- privacy level
- models
- owner
- status
- capacity
- scheduling tags
- allowed workloads
- trust domain

## Pilot Licenses

Suggested fields:

- key or credential reference
- issued to
- permissions
- notes
- allowed castes
- allowed compute classes
- allowed groups
- expiry
- revocation state

## Product Requirement

The contributor page should feel like backstage administration, not like the shared office.
