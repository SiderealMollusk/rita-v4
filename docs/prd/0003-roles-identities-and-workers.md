# 0003 - Roles, Identities, And Workers

## Contributor

A human participant in the collective.

Contributors:

- direct work
- manage teams
- supply compute
- supply or approve pilots

## Soul

A `Soul` is a durable continuity/personality service.

In the current model, a Soul is best understood as:

- a standard chat endpoint
- consulted by a pilot agent at some interval
- carrying durable continuity, preferences, and style

A Soul is not the same thing as a running worker process.

## Caste

A `Caste` defines:

- skills
- tools
- permissions
- system prompt / doctrine
- model class

A Caste is the professional role template.

## Character

A `Character` is:

- `Soul + Caste + Overrides`

It is the recognizable operative identity.

## Agent Instance

An `Agent Instance` is a live embodiment:

- `Character + Compute`
- or `Caste + Compute` for soulless execution

## Court

A `Court` is a contributor-managed aggregate of:

- souls
- castes
- characters
- compute allocations
- pilot licenses
