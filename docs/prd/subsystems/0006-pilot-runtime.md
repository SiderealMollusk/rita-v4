# 0006 - Pilot Runtime

## Purpose

The `Pilot Runtime` is the active reasoning process that inhabits work roles.

It is the layer that consults Souls, applies Castes, and performs work through
the shell and machine core.

## Responsibilities

The pilot runtime should:
- consult Souls when continuity matters
- operate inside the permissions and doctrine of a Caste
- consume context and produce work outputs
- use the Nextcloud shell for visible office behavior
- use the Hive for safe execution and coordination

## Product Role

Pilots are where intelligence shows up.

They are not the same thing as Souls, and they are not the same thing as the
Hive.

## Non-Goals

The pilot runtime is not:
- the durable managerial memory by itself
- the system scheduler
- the compute contribution appliance

## Main Boundaries

The pilot runtime speaks to:
- Souls
- the Hive
- the Nextcloud shell
- model-serving resources, usually through a mana-server-shaped abstraction
