# 0002 - Hive

## Purpose

The `Hive` is the soulless machine core.

It makes the visible office behavior reliable.

## Responsibilities

The Hive owns:
- schedulers
- locks and leases
- queue semantics
- execution policy
- run-state durability
- retries and idempotency
- caste-aware work dispatch

## Key Character

The Hive is intentionally not anthropomorphic.

It is infrastructure and coordination logic, not a personality-bearing worker.

## Non-Goals

The Hive is not:
- a Soul
- a pilot
- a contributor-facing workspace
- a chat surface

## Main Boundaries

The Hive speaks to:
- Nextcloud
- contributor servers
- mana servers
- pilot runtimes

It should remain the place where machine truth is coordinated, not where work is
socially presented.
