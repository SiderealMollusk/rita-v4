# 0004 - Contributor Page

## Purpose

The `Contributor Page` is the backstage UI for contributors.

It is where a trusted contributor manages the collective-facing parts of their
participation.

## Responsibilities

The contributor page should make it easy to:
- register and inspect compute resources
- manage pilot licenses
- inspect usage and budget visibility
- manage teams, souls, castes, and characters
- understand what the contributor is supplying to the collective

## UX Intent

The contributor page should feel like backstage administration.

It should not feel like the office itself.

## Non-Goals

The contributor page is not:
- the shared collaborative surface
- the place where ordinary work happens
- the canonical execution ledger

## Main Boundaries

The contributor page speaks through the contributor server.

It should not bypass the backstage backend and talk to the Hive directly.
